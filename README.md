# Список покупок — Облачное приложение

Приложение для предмета «Основы облачных и туманных технологий».

## Возможности

- **Регистрация и вход** — учётные записи пользователей
- **Несколько списков** — создание, редактирование, удаление, копирование
- **Шаблоны** — «Продукты на неделю», «Пикник», «Домашняя уборка»
- **Совместный доступ** — приглашение других пользователей по имени
- **Категории** — продукты, бытовая химия, молочное, овощи/фрукты
- **Поиск** — по всем спискам
- **Экспорт** — TXT и JSON
- **Печать** — компактный вид для печати
- **История** — последние купленные товары
- **Статистика** — по категориям
- **Настройки** — смена пароля, email и загрузка аватара
- **REST API** — JSON API для интеграций

## Локальный запуск

```bash
pip install -r requirements.txt
python app.py
```

Откройте http://localhost:5000 — сначала зарегистрируйтесь или войдите.

> Схема БД обновляется автоматически при старте (миграции SQLite).

## Развёртывание на Render (бесплатно)

1. Зарегистрируйтесь на [render.com](https://render.com)
2. **New** → **Web Service** → подключите репозиторий GitHub
3. Настройки:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn --bind 0.0.0.0:$PORT app:app` (не `python app.py` — иначе Flask dev-сервер слушает только `127.0.0.1` и Render не видит порт)
   - **Environment:** добавьте `SECRET_KEY` и переменные S3 (см. ниже)
4. **Create Web Service**

Если в логах видно `Running 'python app.py'` — откройте **Settings** → **Start Command** и замените команду на строку с `gunicorn` выше (или очистите поле, если используете `Procfile` / `render.yaml`).

## Подключение S3 для аватаров

1. Создайте S3-совместимое хранилище (bucket), например: `shopping-app`.
2. Создайте доступ (AWS IAM user / ключи) или сервисный аккаунт с правами минимум:
   - `s3:PutObject` (загрузка)
   - `s3:GetObject` (чтение для превью)
   - `s3:DeleteObject` (опционально — удаление старого аватара)
3. (Важно) Для вывода превью приложение использует `presigned URL`, поэтому bucket может быть приватным.
4. На Render (или другом хостинге) добавьте Environment variables, например:
   - `S3_BUCKET_NAME`
   - `S3_REGION` (если нужно)
   - `S3_ENDPOINT_URL` (если не AWS)
   - `S3_ACCESS_KEY_ID` / `S3_SECRET_ACCESS_KEY` или `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY`
5. Откройте в приложении `Настройки` и загрузите аватар — он будет сохранён в S3, а его ключ (`avatar_key`) — в SQLite.
